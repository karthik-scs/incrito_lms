* **Follow-up**: the first pass still let some of YouTube's own chrome bleed through — `controls: 0` turns off the bottom play bar, but YouTube draws its title/channel-name bar and a share icon at the frame's edges independently of that setting, and neither is controllable via a `playerVars` flag. Fixed by cropping: the iframe now renders inside an `overflow-hidden` container at 118% scale (`top/left: -9%`, `width/height: 118%`), pushing those edge-drawn elements outside the visible box while the actual video content still fills the frame correctly. Combined with the iframe staying `pointer-events-none` (so hover/click never reaches YouTube's page to trigger any native overlay in the first place) and `cc_load_policy: 0` added to the existing `playerVars`, every piece the user flagged — title, channel name, YouTube logo, share button, suggested-videos grid — is now suppressed for every YouTube-sourced lesson, not just the bottom control bar. Also added the explicitly requested controls that were still missing: ±30 second skip (`seekTo(currentTime ± 30)`), a real volume slider (`setVolume`, expands on hover next to the existing mute toggle), and a playback-speed menu (`setPlaybackRate`, 0.5×–2×) alongside the existing play/pause, seek bar, and fullscreen.
* **Second follow-up — the real root cause**: the cropping/`pointer-events-none` fix only addresses YouTube's UI *during playback*; it did nothing about the branded title/channel-name/giant-red-play-button poster YouTube renders as the embed's *static initial state*, before any video has actually started — that poster is baked into the embed's own markup, not a hover-triggered overlay, so no `playerVars` flag or CSS crop removes it (cropping only pushes the title bar partway off-frame; the centered play button and "Watch on YouTube" text sit too close to center to crop away without losing real video area). First fix attempt: hide the iframe (`opacity-0`) only until the first play click, on the assumption YouTube never re-shows that branded card once playback has begun once.
* **Third follow-up**: that assumption was wrong — a follow-up screenshot showed the exact same branding (title, channel name, "More videos") still visible *mid-video*, on pause, not just before the first play. YouTube shows this branded idle card any time the player **isn't actively playing** — paused or ended, not just "never started" — and there's still no parameter to suppress it. The real fix: stop keying visibility off "has it ever played" and key it off "is it playing right now." The iframe now sits at `opacity-0` (always mounted, API still controls it) whenever `playing` is false — covers before-first-play, every pause, and the end-of-video state alike — and this app's own cover (now using the lesson's `thumbnailUrl` as a background image when one's set, plain dark background otherwise) is shown in its place. The cover only disappears while a frame is actually being decoded and displayed by YouTube, which is the one state it has never been observed to brand. Pausing now means "this app's cover reappears," not "YouTube's branding reappears" — the cosmetic cost is that a paused video shows this app's static cover instead of the frozen last frame (cross-origin iframe content can't be canvas-captured for a real freeze-frame), which is the accepted trade-off for the alternative being literally unfixable YouTube branding.
* **Lesson thumbnails**: added `Lesson.thumbnailUrl` (migrated as `phase9_lesson_thumbnail`) — the admin lesson form (`/admin/courses/[slug]`) gained a "Thumbnail" URL field for `VIDEO`-type lessons, with a live `<img>` preview right in the modal (silently hides itself via `onError` if the URL doesn't actually resolve to an image, rather than showing a broken-image icon).
